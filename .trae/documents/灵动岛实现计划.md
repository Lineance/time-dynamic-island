# 灵动岛（Dynamic Island）Windows 后台程序实现计划

## 1. 项目概述

创建一个运行在 Windows 11 后台的灵动岛程序，在屏幕上方显示时间，具备以下特性：
- 椭圆形时间显示
- 自适应系统主题
- 鼠标交互（悬停放大、左键设置时间、右键切换格式）
- 开机自启动
- 极低内存占用（<5MB）
- 规避游戏反作弊检测

## 2. 技术选型

### 2.1 编程语言：C++
**选择理由**：
- 原生 Windows API 支持，无需运行时环境（.NET/Java 等）
- 内存占用极低（预计 2-5MB）
- 编译为本地机器码，执行效率高
- 更容易规避反作弊软件检测
- 开机自启动实现简单（注册表/启动文件夹）

### 2.2 技术栈
- **GUI 框架**：Win32 API（原生，零依赖）
- **渲染方式**：GDI+（支持透明度、抗锯齿）
- **系统主题检测**：Windows Registry API
- **时间格式**：C++17 `<chrono>`
- **构建工具**：CMake + Visual Studio 2022
- **打包工具**：Inno Setup（可选）

## 3. 实现步骤

### 3.1 项目初始化
1. 创建 CMake 项目结构
2. 配置 Visual Studio 2022 编译器设置
3. 设置 C++17 标准
4. 链接必要的 Windows 库（user32, gdi32, gdiplus, shell32）

### 3.2 核心模块设计

#### 模块 1：窗口管理（WindowManager）
- 创建无边框、透明背景的顶层窗口
- 窗口位置：屏幕顶部居中
- 窗口样式：WS_POPUP | WS_EX_TOPMOST | WS_EX_LAYERED | WS_EX_TRANSPARENT
- 实现窗口大小调整动画（悬停放大效果）

#### 模块 2：渲染引擎（Renderer）
- 初始化 GDI+
- 绘制椭圆形背景（带抗锯齿）
- 绘制时间文本（居中、抗锯齿）
- 颜色方案：
  - 浅色主题：白色背景 + 黑色文字
  - 深色主题：黑色背景 + 白色文字
- 实现透明度混合

#### 模块 3：时间服务（TimeService）
- 使用 C++17 `<chrono>` 获取系统时间
- 支持 24 小时制和 12 小时制格式化
- 定时器更新（每秒刷新）
- 格式切换逻辑

#### 模块 4：主题监听（ThemeMonitor）
- 监听注册表变化：`HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Themes\Personalize`
- 检测 `AppsUseLightTheme` 键值
- 自动切换颜色方案

#### 模块 5：交互处理（InteractionHandler）
- 鼠标跟踪（WM_MOUSEMOVE, WM_MOUSELEAVE）
- 悬停检测与放大动画
- 鼠标点击处理：
  - 左键：调用 `control.exe timedate.cpl` 打开时间设置
- 右键菜单：切换 24/12 小时制

#### 模块 6：反作弊规避（AntiCheatEvasion）
- **关键措施**：
  1. 窗口样式伪装：设置为工具窗口（WS_EX_TOOLWINDOW）
  2. 不注入任何游戏进程
  3. 不使用钩子（Hook）技术
  4. 签名验证：使用有效证书签名（可选）
  5. 进程名称：使用中性名称（如 `SystemTrayHelper.exe`）
  6. 避免内存修改行为
  7. 不在游戏运行时主动渲染（检测全屏游戏）

#### 模块 7：开机自启动（StartupManager）
- 方案 A：注册表启动项
  - `HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run`
- 方案 B：启动文件夹快捷方式
  - `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`
- 提供配置选项（可通过配置文件开关）


## 5. 内存优化策略

1. **资源管理**：
   - GDI+ 对象及时释放（Brush、Pen、Font）
   - 使用 RAII 模式管理资源

2. **渲染优化**：
   - 仅在时间变化时重绘（每秒 1 次）
   - 悬停时使用双缓冲减少闪烁

3. **线程模型**：
   - 单线程消息循环
   - 定时器驱动更新（无需额外线程）

4. **目标内存占用**：
   - 空闲时：< 2MB
   - 渲染时：< 5MB

## 6. 反作弊规避详细说明

### ACE 检测机制分析
ACE（Anti-Cheat Expert）可能检测：
- 窗口覆盖（Overlay）
- 进程注入
- API Hook
- 内存修改
- 异常进程行为

### 规避策略
1. **窗口样式**：
   - 使用 `WS_EX_TOOLWINDOW`（工具窗口，不在 Alt+Tab 显示）
   - 设置 `WS_EX_TRANSPARENT`（鼠标事件穿透，非游戏窗口）

2. **进程行为**：
   - 不注入任何 DLL 到其他进程
   - 不使用 SetWindowsHookEx 等钩子 API
   - 不读取/修改游戏内存

3. **签名与白名单**：
   - 使用有效代码签名证书（可选，商业发布时）
   - 进程名称中性化

4. **游戏检测**：
   - 检测全屏应用运行时暂停渲染
   - 检测游戏进程列表（可配置黑名单）

## 9. 风险评估

### 技术风险
- **GDI+ 性能**：可能在低配设备上卡顿 → 优化渲染频率
- **主题检测延迟**：注册表监听可能有延迟 → 轮询 + 事件结合
- **ACE 更新**：反作弊规则可能变化 → 持续监控与更新

### 法律风险
- **反作弊规避**：可能违反游戏服务条款 → 仅提供技术实现，用户自行承担风险
